The need for testing

There was a study conducted by the National Institute of Standards and Technology (NIST) in 2002, which reported that software bugs cost the U.S. economy around $60 billion annually.

The need for testing

  • To check if the software is functioning as per the requirements
  • There will not be just one device or one platform to run your software
  • The end users sometimes perform an action as a programmer that you never expected, and these actions might result in defects. To catch these defects, we need testing

Types of testing

Any product, website, or utility, should be very well tested before it is launched.

The life cycle of TDD

The life cycle of TDD

  • Write a test
  • Run the test and see if the test fails
  • Write minimal production code that passes
  • Run all tests
  • The cleanup code
  • Repeat

TDD microcycle

The TDD life cycle is also defined as Red-Green-Refactor, which is also called as the microcycle.

The life cycle of TDD

  • Write a test that fails at the start (Red)
  • Write just enough code to pass the test (Green)
  • Refactor the code (Refactor)
